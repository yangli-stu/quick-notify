深入理解 Quick-Notify 的整体设计和核心组件

整体架构图

核心组件

StompWebsocketInterceptor

Token 认证拦截器，在 CONNECT 时验证用户身份，支持连接数限制（每用户最多 10 个）。

StompWebSocketHandler

消息处理器，负责消息发送、ACK 追踪、定时重试。核心逻辑所在。

NotifyEventListener

Redis Pub/Sub 事件监听器，处理集群环境下跨节点消息转发。

NotifyManager

消息管理器，统一编排消息持久化和发布，负责事务管理。

数据库设计

SQL 
CREATE TABLE notify_log (
    id VARCHAR(64) PRIMARY KEY,           -- 雪花 ID
    type VARCHAR(64) NOT NULL,             -- 消息类型
    receiver VARCHAR(64) NOT NULL,         -- 接收者
    data TEXT,                             -- 消息内容 (JSON)
    viewed BOOLEAN DEFAULT FALSE,          -- 是否已读
    created BIGINT,                        -- 创建时间
    last_modified BIGINT,                  -- 修改时间
    version BIGINT DEFAULT 0,              -- 乐观锁

    INDEX idx_receiver_viewed_created (receiver, viewed, created)
);
