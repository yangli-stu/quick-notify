概述

Quick-Notify 是一个基于 Spring Boot 3 + WebSocket STOMP + Redis 的企业级实时消息推送系统。

特性

  • Starter 模式：一行依赖即可接入
  • ACK 确认机制：确保消息可靠送达
  • 集群支持：基于 Redis Pub/Sub
  • 多设备同步：每设备独立确认
  • 消息持久化：历史消息存储
  • Token 认证：安全可靠

技术栈

  • Spring Boot 3.2.11
  • Java 21
  • WebSocket + STOMP + SockJS
  • Redis (Redisson)
  • MySQL / H2

快速开始

环境要求

  • JDK 21+
  • Maven 3.8+
  • Redis 6+

1. 添加依赖

XML 
<dependency>
  <groupId>io.stu</groupId>
  <artifactId>quick-notify-spring-boot-starter</artifactId>
  <version>0.0.1-SNAPSHOT</version>
</dependency>

2. 配置 Redis

YAML 
# application.yml
spring:
  datasource:
    url: jdbc:h2:mem:quicknotify
  sql:
    init:
      mode: always

redisson:
  single-server-config:
    address: redis://127.0.0.1:6379

3. 发送消息

Java 
@Autowired
private NotifyManager notifyManager;

public void sendNotification() {
    NotifyMessageLog message = NotifyMessageLog.builder()
        .receiver("user123")
        .type("ORDER_STATUS")
        .data(Map.of(
            "orderId", "ORD_001",
            "status", "PAID"
        ))
        .build();

    notifyManager.saveAndPublish(message);
}

4. 前端订阅

JavaScript 
const stompClient = Stomp.over(new SockJS('/stomp-ws'));

stompClient.connect(
    { Authorization: 'Bearer your-token' },
    () => {
        stompClient.subscribe('/user/queue/msg', (msg) => {
            const data = JSON.parse(msg.body);
            console.log('收到消息:', data);
            stompClient.send('/app/ack', {}, data.id);
        });
    }
);

STOMP 路径

WebSocket 连接流程

WebSocket 连接需要先携带 Token 认证，连接成功后再订阅消息：

JavaScript 
// Step 1: 创建 STOMP 客户端
const stompClient = Stomp.over(new SockJS('/stomp-ws'));

// Step 2: 连接 WebSocket（dev 模式 token 直接作为用户ID）
stompClient.connect(
    { Authorization: 'testuser' },

    // Step 3: 连接成功回调 - 在此处订阅消息
    (frame) => {
        console.log('连接成功');

        // 订阅个人消息队列
        stompClient.subscribe('/user/queue/msg', (msg) => {
            const data = JSON.parse(msg.body);
            console.log('收到消息:', data);

            // 发送 ACK 确认
            stompClient.send('/app/ack', {}, data.id);
        });
    },

    // 连接失败回调
    (error) => { console.error('连接失败', error); }
);

路径说明

路径 类型 说明
/user/queue/msg 订阅 接收个人消息（自动路由到 /user/{userId}/queue/msg）
/topic/messages 订阅 接收广播消息（所有订阅者都能收到）
/app/sendMessage 发送 客户端发送消息，路由到 @MessageMapping
/app/ack 发送 消息确认（携带 messageId）

认证机制

客户端连接时需要在 CONNECT 帧的 Authorization header 中携带 Token：

JavaScript 
stompClient.connect(
    { Authorization: 'Bearer eyJhbGciOiJIUzI1NiJ9...' },
    (frame) => { // 连接成功 },
    (error) => { // 连接失败 }
);

自定义认证

继承 StompWebsocketInterceptor 并重写 extractUserId 方法：

Java 
public class MyAuthInterceptor extends StompWebsocketInterceptor {

    @Override
    protected String extractUserId(StompHeaderAccessor accessor) {
        String token = accessor.getNativeHeader("Authorization").get(0);
        return jwtService.extractUserId(token);
    }
}

ACK 确认机制

Quick-Notify 内置消息确认机制，确保消息可靠送达：

  • 发送消息时创建 ACK 记录
  • 客户端收到消息后发送确认
  • 未确认的消息自动重试（最多 12 次）
  • 60 秒 TTL 自动清理

客户端 ACK

注意：以下代码需要在 stompClient.connect() 的成功回调中执行：

JavaScript 
// 连接成功后订阅（完整示例见上文）
stompClient.subscribe('/user/queue/msg', (msg) => {
    const data = JSON.parse(msg.body);
    // 处理业务逻辑
    processMessage(data);
    // 发送 ACK 确认
    stompClient.send('/app/ack', {}, data.id);
});

集群部署

Quick-Notify 支持多节点集群部署，通过 Redis Pub/Sub 实现跨节点消息转发：

YAML 
# Redis Sentinel 高可用
redisson:
  sentinel-servers-config:
    sentinelAddresses:
      - redis://127.0.0.1:6379
      - redis://127.0.0.1:6378
      - redis://127.0.0.1:6377
    sentinelMasterName: my-master

NGINX 配置

Nginx 
upstream backend {
    server app1:8080;
    server app2:8080;
}

server {
    location /stomp-ws {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
