为什么选择 Quick-Notify？

开箱即用的企业级解决方案，让实时消息推送变得简单

Starter 模式

一行依赖即可接入，零配置启用。自动配置所有组件，开箱即用。

ACK 确认机制

自动重试机制确保消息可靠送达，解决网络抖动、客户端离线等场景的消息丢失问题。

集群支持

基于 Redis Pub/Sub 实现跨节点消息转发，多节点水平扩展无忧。

安全认证

Token 鉴权，每用户最多 10 个连接，连接数限制防止资源耗尽攻击。

多设备同步

同一用户多设备独立 ACK 追踪，消息同步不遗漏。

消息持久化

支持历史消息存储与查询，消息记录完整可追溯。

快速开始

只需三步，即可在你的项目中启用实时消息推送

1. 添加依赖

XML 
<dependency>
  <groupId>io.stu</groupId>
  <artifactId>quick-notify-spring-boot-starter</artifactId>
  <version>0.0.1-SNAPSHOT</version>
</dependency>

2. 发送消息

Java 
@Autowired
private NotifyManager notifyManager;

notifyManager.saveAndPublish(
    NotifyMessageLog.builder()
        .receiver("user123")
        .type("ORDER_STATUS")
        .data(orderData)
        .build()
);

3. 前端订阅

JavaScript 
stompClient.subscribe(
    '/user/queue/msg',
    (msg) => {
        const data = JSON.parse(msg.body);
        stompClient.send('/app/ack', {}, data.id);
    }
);
